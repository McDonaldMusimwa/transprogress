1. Reset the Expectation (This Matters)

Your goal right now is NOT:

"Be a senior-level contributor"

Your goal IS:

"Be reliably useful and improve every week."

Managers don't expect juniors to be fast. They expect juniors to:

  • Communicate
  • Ask good questions
  • Make steady progress
  • Not break production

If you do those well, you're already performing.

3. What "Good Performance" Looks Like for a Frontend Dev

A. You understand the codebase flow

Not all code ‚Äî just:

  • How data enters the app
  • How state is managed
  • How API calls are made
  • How errors/loading are handled
Action:

B. You ship small things confidently

You don't need big wins yet. Aim for:

  • Fixing bugs
  • Improving edge cases
  • Cleaning small parts
  • Writing safer TypeScript

Consistency > flashiness.

C. You communicate progress clearly

This is underrated but huge.

Instead of:

5. How to Ask for Help Without Looking Weak

Never say:
"I'm stuck."
Say:
"I understand X and Y, but I'm unsure about Z. Would you recommend approach A or B?"

That shows:

  • You tried
  • You're thinking
  • You respect their time

This alone changes how people perceive you.

6. Backend & AWS ‚Äî Don't Panic

You don't need to master AWS right now.

Focus on:

  • What services you actually use (e.g. S3, Lambda, API Gateway)
  • What problem each one solves
  • How your frontend interacts with it

High-level understanding is enough early on.

7. One Mindset Shift That Helps Massively

Instead of:
"I'm not performing"
Ask:
"What would slightly better performance look like next week?"

Examples:

  • Understanding one more part of the system
  • Finishing tickets with fewer revisions
  • Asking clearer questions
  • Catching one bug earlier

Small wins compound fast.
